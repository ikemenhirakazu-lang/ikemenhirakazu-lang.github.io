# ブログ立ち上げ 設計書

作成日: 2026-07-22
ステータス: 承認済み(実装計画へ移行前)

## 1. プロジェクトの目的

「土木一筋・非IT・49歳のおじさんが、Claude Codeで何をどこまでできるのか」を、
初心者の挑戦・失敗・気づきごと記録する **実況・実験ログ型ブログ** を立ち上げ、
継続的に運営する。将来的にアフィリエイトでの収益化を目指すが、当面は継続を最優先とする。

### コンセプトの核

- 前職・現場代理人としての「人に動いてもらって成果を出す」視点を、AIエージェント活用に重ねる
- 技術解説ではなく **実体験の記録**。失敗も戸惑いもそのまま書く
- 文体は「丁寧だけどお調子者」。調子に乗っては四苦八苦する様子を隠さない
- ポジション: 非IT・地方・40代後半の属性で生々しく発信するブログは少ない。そこが独自性

### 参考にした情報源

ASP公式(afb)×ラッキーマイン氏の対談動画(2026年2月)の要点を反映:

- ゼロからAIに書かせた記事は皆同じになる。**実体験×AI** が価値の源泉 → 本ブログのコンセプトと一致
- ASP審査はコンテンツが溜まってから申請する(投稿ゼロ・非公開は落ちる)
- ブログ単独のSEO勝負は激戦。SNS(特にInstagram)が主戦場 → ブログを母艦にSNS導線を後日追加
- セルフB等の自己アフィリエイトは「体験→記事化」の素材源として使う
- 稼げない人は「アカウント作成から」始める。正しくはジャンル選定→調査→方針→開設の順

## 3. フォルダ構成

**ネタ帳(非公開)とサイト本体(公開)を分離する。**

理由:
1. OneDriveとgitは同期衝突を起こしやすい
2. GitHub Pagesの無料公開はリポジトリが全世界公開になるため、未公開ネタを同居させない
3. ネタ帳はOneDriveが自動バックアップし、スマホのOneDriveアプリから直接貼り付けできる

```
Desktop\AIブログ\              ← OneDrive同期。非公開のネタ帳・作業場
├── docs\                      ← 設計書などのドキュメント
├── ideas\
│   ├── inbox\                 ← Plaud文字起こしの貼り付け先(YYYY-MM-DD-題名.md)
│   └── processed\             ← ネタ化が済んだメモ
├── topics.md                  ← 記事ネタ一覧(候補/下書き中/公開済みを管理)
└── drafts\                    ← 記事の下書き

C:\Users\ikeme\projects\blog\  ← サイト本体。git管理、GitHubで公開
```

## 4. ideas/ 運用ルール

1. **録る**: 日々の気づきをPlaudで録音、アプリで文字起こし(無料枠300分/月)
2. **貼る**: テキストを `ideas\inbox\YYYY-MM-DD-題名.md` に貼り付け(スマホのOneDriveアプリからでも可)
3. **任せる**: 週1回程度「inboxを整理して」とClaude Codeに指示
   → メモを読んでネタを `topics.md` に追記し、元ファイルを `processed\` へ移動
4. **膨らませる**: `topics.md` からネタを選び「下書きにして」と指示 → `drafts\` に下書き生成

最初は手動指示で回し、型が固まったら定型作業をスキル化する。この進化の過程も記事ネタにする。

## 5. サイトの初期構成

### 固定ページ(ASP審査対策として最初に用意)

- 運営者情報(プロフィール)
- プライバシーポリシー
- お問い合わせ(Googleフォーム埋め込みで可)

### 記事の仕組み

- 記事はMarkdownファイル。frontmatterで日付・タイトル・タグ等を管理
- **PR表記の自動化**: frontmatterに「アフィリエイト含む」フラグを立てると
  記事冒頭に「広告/PR」表記が自動表示される(ステマ規制・景品表示法対応)
- 執筆フロー: 実体験メモ→構成→草稿(言葉の質が勝負の箇所はFable 5活用)→本人の言葉で修正→公開

## 6. 「第0話」構成案

仮タイトル: 「49歳・土木一筋・非IT。ゲームしかしてこなかったおじさんが、AIに仕事を頼み始めた」

1. **つかみ**: この記事をClaude Codeと書いている実況から(「調子に乗ってブログまで始めてしまいました」)。
   「第0話」の由来ネタ: プログラミングの世界は数を0から数える文化がある → 「IT入門したので話数も0から数えます」。
   第0話=挑戦前の自己紹介回、第1話以降=実際の挑戦の記録、という位置づけ
2. **自己紹介**: 土木20年超(積算・施工管理)、プライベートはゲームのみ、ITは門外漢
3. **きっかけ**: Claude Codeとの出会い、何にワクワクしたか
4. **核となる視点**: 現場代理人=「人に動いてもらって成果を出す」仕事。AIエージェントも同じという仮説
5. **正直な棚卸し**: 今できること/できないこと
6. **無謀な宣言**: ブログ・せどり・ダイエットアプリ・競馬予想・FX・株ペーパートレード…全部着手済みという お調子者ぶりの披露と「全部記録する」宣言
7. **締め**: 同じ非ITのおじさん世代への語りかけ

## 7. ロードマップ

### 今日(構築フェーズ)
1. ネタ帳フォルダ一式を作成
2. `C:\Users\ikeme\projects\blog` にAstroブログを構築、手元で表示確認
3. 固定3ページの雛形と第0話の草稿を作成
4. GitHubリポジトリ作成、GitHub Pagesで仮公開(この時点では github.io ドメイン)

### 公開準備フェーズ(〜数日中)
5. ブログ名・ハンドルネーム・ドメイン名を決定(Xのアカウント名も同時に決める)
6. 独自ドメイン取得・接続(SEO評価がリセットされるため、本格公開前に済ませる)
7. 第0話を仕上げて公開
8. **Xアカウントを開設し、ブログと同時に運用開始**
   - 役割: 記事公開の告知+日々の短い実況ポスト(「今日Claude Codeにこう頼んだらこうなった」)
   - 理由: 新規ドメインは検索流入が育つまで数か月かかるため、初期の読者獲得はSNS経由が生命線。
     文字中心のXは手間が小さく、実況コンセプトとの相性も良い

### 育成フェーズ(〜1か月目安)
9. 記事を3〜5本公開(各1,000字以上。せどり・各プロジェクトの体験記)。Xで告知・実況を継続
10. ASP申請(1社目はA8.net、次にafb)。セルフBで体験→記事化のサイクル開始
11. アクセス解析(Search Console等)を導入 — これも記事ネタ

### 将来(収益・拡張フェーズ)
- Instagram導線の追加(ASPデータでは支払いの過半数を占める本命。ただし画像・動画素材が必要なため、記事と素材が溜まってから)
- ideas取込やX投稿の自動化(sns-automationの知見を活用)
- 収益発生後: 勤め先の副業規定の確認、確定申告(住民税)の検討
